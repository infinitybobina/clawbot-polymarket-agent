# 90-дневный бэктест и симуляция цен

## Почему 30 дней — рабочие, а 90 — пока плохо?

**Да, это связано с симуляцией цен.**

1. **Один сценарий** — цены генерируются с фиксированным `price_seed=42`. Один и тот же путь цен каждый раз. Первые 30 дней этого сценария могут быть удачными для стратегии, следующие 60 — менее (другая «погода» на симулированном рынке).

2. **Упрощённая модель** — тренды/флэт/спайки без привязки к реальным событиям и ликвидности. На длинном горизонте такое поведение может быть нереалистичным (слишком сильный дрейф или просадки).

3. **Мало сделок на длинном периоде** — те же ~9 сделок на 90 дней; позиции дольше живут в симулированной волатильности, итоговые Sharpe и прибыль на 90 днях получаются хуже.

## Что можно сделать

- **Рабочие показатели брать с 30 дней** — разумно считать их допустимыми для текущей стратегии и параметров.
- **90 дней проверять на разных сценариях** — задавать разный `price_seed` и смотреть разброс (устойчивость стратегии):

  ```bash
  python -c "
  from backtest import Backtester
  import logging; logging.basicConfig(level=logging.WARNING)
  for seed in [42, 100, 200]:
      r = Backtester(days=90, n_markets=12, min_yes_edge=0.08, momentum_threshold=0.02, price_seed=seed).run()
      print(f'seed={seed}: Sharpe={r[\"sharpe\"]:.3f}, Profit%={r[\"profit_pct\"]:.2f}, DD%={r[\"max_drawdown_pct\"]:.2f}')
  "
  ```

- **Итоговую оценку по 90 дням — по реальным данным** (исторические OHLCV с Polymarket/Gamma), когда они будут подключены; симуляция даёт только ориентир.

## Параметр price_seed

В `Backtester` добавлен параметр **`price_seed`** (по умолчанию 42). Разные значения дают разные траектории цен при той же логике симуляции, так что можно оценить, насколько 90-дневные результаты зависят от сценария.
