# Как получать техническую информацию о работе ClawBot

## 1. Файловый лог (основной канал)

**Файл:** `clawbot_run.log` (в корне проекта)

- Пишется при каждом запуске (дозапись).
- Уровень: **INFO** — все ключевые шаги алгоритма.
- Формат: `%(asctime)s [%(levelname)s] %(name)s: %(message)s`

**Что видно по шагам:**

| Этап | Что логируется |
|------|----------------|
| **Datafeed** | Кол-во загруженных politics/crypto рынков, топ кандидатов |
| **Portfolio** | Загрузка/сохранение: balance, positions, realized_pnl |
| **Cooldown** | Закрытия по SL/TP, счётчики cooldown по рынкам |
| **Кандидаты** | Исключения: held, SL/TP cooldown, фильтр YES ≥ max_entry или YES < 0.02 |
| **Strategy** | 0 кандидатов → skip LLM; иначе вызов LLM, кол-во сигналов |
| **Risk** | entry/sl/tp, отказы (ENTRY_TOO_HIGH, INVALID_SL_TP, EXCEEDED_*), одобренные ордера |
| **Volume cap** | Подрезка размера по объёму/глубине рынка (политика/крипто) |
| **Paper trader** | FILLED (размер, цена), CLOSED (PnL), unrealized PnL по позициям |
| **Telegram** | Успех/неуспех отправки |

При запуске из Планировщика задач консоль недоступна — смотрите **только** `clawbot_run.log`.

---

## 2. Консоль (при ручном запуске)

При `python main.py` те же сообщения дублируются в **stdout** (консоль). Дополнительно в консоль выводятся:

- `Portfolio after trades:` и метрики
- `FINAL RESULTS:` (JSON с балансом, позициями, PnL)

Удобно для быстрой отладки «на месте».

---

## 3. Telegram

В чат уходит сводка после каждого цикла:

- Режим (SIMULATION), баланс, кол-во позиций, realized PnL
- Закрытия по SL/TP (если были) с PnL
- Новые сделки (LIVE): рынок, категория, EV, экспозиция
- Итог: Signals / Approved

Нужны `TELEGRAM_TOKEN` и `TELEGRAM_CHAT_ID` в `.env`. Полный список переменных окружения — **[ENV.md](ENV.md)** и **.env.example** в корне репо.

---

## 4. Включить подробный лог (DEBUG)

В `config.py` в **PROD_CONFIG** задано поле **`"log_level": "INFO"`**. Для отладки смените на:

```python
"log_level": "DEBUG",
```

Перезапустите бота — в `clawbot_run.log` и в консоль пойдут и сообщения уровня DEBUG от всех модулей (datafeed, strategy, risk, paper_trader и т.д.), где такой вывод реализован. Без правки кода, только конфиг.

---

## 5. Состояние между запусками

- **portfolio_state.json** — баланс, позиции, cumulative_realized_pnl (что «помнит» бот к следующему запуску).
- **sl_cooldown.json**, **tp_cooldown.json** — какие рынки в cooldown и на сколько запусков.

По ним можно понять, почему на следующем запуске не открывается позиция (уже в позиции или в cooldown).

## Кратко

| Вопрос | Где смотреть |
|--------|--------------|
| Что происходит внутри алгоритма по шагам? | `clawbot_run.log` (и консоль при ручном запуске) |
| Почему нет кандидатов / нет сигналов? | Лог: блоки «Excluded … candidates left», «LLM: 0 candidates», «Risk rejected» |
| Почему ордер отклонил риск? | Лог: `Risk: ENTRY_TOO_HIGH`, `INVALID_SL_TP`, `EXCEEDED_*` |
| Какие сделки и по какой цене? | Лог: `FILLED`, `CLOSED`, `PnL` от `paper_trader` |
| Сводка в телефоне | Telegram после каждого цикла |
| Состояние к следующему запуску | `portfolio_state.json`, `*_cooldown.json` |
